_$ Terminal 101

"The terminal is to hacking as the hammer is to woodwork." --Some Guy I Made Up

By far one of the most important applications on Linux, Mac, and Windows, the terminal gives users a means of issuing commands directly to the computer via a simple command-line. A user can do everything in terminal that they can do from the desktop GUI, including creating and deleting files and directories, sending mail, setting up network connections, formatting partitions, et c, just faster and more efficiently.

Most important for our purposes is that the terminal can do some very powerful things the desktop GUI just can't do. Anything you do in terminal once, can be automated and done as many times as you want whilst you have a sandwich. It also gives us access to Python, the Chuck Norris of hacking Swiss Army knives. Seriously, Python can do damn near anything.

I could go on, but if you want to explore hacking you are going to need to know the command-line inside out.

Lets get into it.


Getting Started

First, the vocabulary:

In the days when dinosaurs roamed the Earth and I was but a pimply CS student, a terminal was a physical object -- a chunky twenty five pound mass of beige and black plastic the size of a microwave oven that connected to a processor somewhere and allowed one to type in commands on a lovely mechanical keyboard and see the results on a crappy monochrome green CRT.

A "terminal emulator" is a bit of software that mimics the functionality of a physical terminal, it takes commands and shows output; when we speak of a "terminal" today, we are actually referencing a "terminal emulator" like konsole, qterminal, or my personal favorite, terminator. I pray the Internet Terminology Police will grant me leave to not type "terminal emulator" a thousand times.

OK, disambiguation achieved; now on to shells. If the terminal is the GUI window that you see on the screen, the shell is the software that interprets and executes the various commands that you type in the terminal. The shell is a command-line interpreter or command prompt, and a terminal is a wrapper program that runs a shell and allows us to enter commands. Some examples of shells would be zsh, fish, powershell, and my personal favorite, bash.

So, to recap: terminal emulator program + shell = terminal

Next, Opening Terminal

There are varying ways of accessing command line, depending on what operating system you use.

  • Linux: You can open Terminal directly by pressing CtrlaltT or you can launch it by clicking the “Dash” icon or pressing the key and typing “terminal” in the search box and opening it. This should open up an app with a black background. When you see your username followed by a dollar sign, you’re ready to start using the command line.

  • MacOS: Open spotlight search by hitting command and the space bar and type in “terminal”. Select the application called terminal and press the return key. This should open up an app with a black background. When you see your username followed by a dollar sign, you’re ready to start using command line.

  • Windows: This is will only work if you have WSL or WSL2 (Windows Subsystem for Linux) installed on your Windows 10 machine. I'll link how to do that here. To verify you have WSL installed, open the Start menu by clicking or pressing and typing WSL. You should see WSL Run command -- which will launch the terminal when clicked. When you see your username followed by a dollar sign, you’re ready to start using command line.



Basic Commands

Now that we are in a terminal, lets learn some basic commands. We will start by figuring out who and where we are on this machine using whoami (who am I) and pwd (Print Working Directory). 

              
                    eric㉿ServerOne─$ whoami
                    eric
                    eric㉿ServerOne─$ pwd
                    /mnt/c/Users/eric

Whoami told us that we are logged on as "eric", and Print Working Directory showed us that we were in /mnt/c/Users/eric, our home directory.
