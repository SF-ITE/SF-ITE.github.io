Linux Terminal 101

"The Linux terminal is to hacking as the hammer is to woodwork."

A terminal, or more properly a "terminal emulator", is just a text-based interface used to control a computer. It's just one of the tools provided to users for accomplishing tasks, but it's widely considered the most efficient method available.

Basics

First, the vocabulary:

In the days when dinosaurs roamed the Earth and I was but a pimply CS student, a terminal was a physical object -- a chunky twenty five pound mass of beige and black plastic the size of a microwave oven. It connected to a processor(s) somewhere and allowed one to type in commands on a lovely mechanical keyboard and see results on a crappy monochrome CRT.

A "terminal emulator" is a bit of software that mimics the functionality of a physical terminal, it takes commands and shows output; when we speak of a "terminal" today, we are actually referencing a "terminal emulator" like konsole, qterminal, or my personal favorite, terminator. I pray the Internet Terminology Police will grant me leave to not type "terminal emulator" a thousand times.

OK, disambiguation achieved; now on to shells. If the terminal is the GUI window that you see on the screen, the shell is the software that interprets and executes the various commands that you type in the terminal. The shell is a command-line interpreter or command prompt, and a terminal is a wrapper program that runs a shell and allows us to enter commands. Some examples of shells would be zsh, fish, powershell, and my personal favorite, bash.

So, to recap: terminal emulator program + shell = terminal

Secondly, Agnosticism

For our purposes, we are just going to use gnome-terminal to run bash -- these are the defaults on Ubuntu and many other Linux distributions. This is as vanilla as it gets and what I show you in this configuration should work perfectly in fancier terminal/shell combos.

    Terminal Shortcuts

  • Auto-complete file and folder names Tab
  • Clear the terminal screen Ctrl+A
  • Exit the current shell Ctrl+D
  • Kill the running process Ctrl+C
  • Suspend running process Ctrl+Z
  • Copy Ctrl+Shift+C
  • Paste Ctrl+Shift+P
  • Last command (forward listing) Up Arrow
  • Last command (reverse listing) Down Arrow

    Change Directory

  • whoami Show your user ID
  • pwd Show your working directory
  • cd.. Go up one directory
  • cd../.. Go up two directories
  • cd- Back to the previous directory
  • cd Home directory
  • cd/ Root of the drive
  • echo $PATH List directories on the path
  • cd/foo Change directory to /foo
